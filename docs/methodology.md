# AB实验标准流程与方法论

---

## 一、实验设计阶段

### 1.1 明确实验目标

**做什么：** 定义要验证的假设和核心指标。

**方法：**
- 用一句话写清楚：如果我们做X改变，Y指标会变好/变差
- 区分**核心指标**（决策依据，如转化率）和**护栏指标**（不能恶化，如页面加载时长、跳出率）

**原理：** 没有明确假设的实验是"钓鱼实验"——先跑再看什么显著，必然导致多重比较问题，假阳性率膨胀。先定假设再检验，才能控制第一类错误率。

### 1.2 确定MDE（最小可检测效应）

**做什么：** 回答"多大的提升才值得检测？"

**方法：**
- 从业务角度定：1%相对提升是否有价值？5%呢？
- 从成本角度定：上线新方案的开发/运维成本，需要多少增量收益才能覆盖？

**原理：** MDE直接决定所需样本量。MDE越小→需要样本量越大→实验周期越长。MDE不是统计问题，是业务判断问题。设太大可能漏掉有价值的改进，设太小实验成本过高。

### 1.3 计算所需样本量

**做什么：** 根据基线指标、MDE、显著性水平α、功效1-β，计算每组最小样本量。

**公式（双比例场景）：**

1. 设基线转化率p₁，MDE对应p₂
2. Cohen's h = 2·arcsin(√p₁) - 2·arcsin(√p₂)
3. 每组样本量 n = (z_{α/2} + z_β)² / h²

**常用参数默认值：** α=0.05（双尾），1-β=0.80

**原理：**
- α控制假阳性（没效果说有效果的概率）
- β控制假阴性（有效果说没效果的概率）
- 样本量越大→标准误越小→检测小效应的能力越强→功效越高
- 功效不足的实验即使真实效应存在也检测不出来，p>0.05不能说明"没效果"，只能说明"证据不足"

### 1.4 分流机制设计

**做什么：** 确定如何将用户随机分配到对照组和实验组。

**方法：**
- **哈希分流：** 对user_id做哈希取模，保证同一用户始终在同一组
- **多层正交实验：** 多个实验同时跑时，用正交表保证实验间互不干扰
- **分流比例：** 一般50:50，早期探索阶段可70:30或90:10降低风险

**原理：** 随机化是AB实验的根基——它确保两组在实验前是同质的（除展示页面不同外），这样观测到的差异才能归因为页面效果。如果分流不随机（比如活跃用户更容易被分到某组），就会引入选择偏差。

### 1.5 实验周期规划

**做什么：** 确定实验跑多久。

**方法：**
- 根据所需样本量÷日均流量计算最低天数
- 必须覆盖完整周期（至少包含完整的7天，覆盖工作日+周末）
- 考虑新奇效应：新功能上线初期用户因新鲜感行为变化，需等效应消退
- 考虑学习效应：用户需要时间适应新设计，短期内效果可能偏负面

**原理：** 用户行为有周期性（工作日vs周末、白天vs晚上），不覆盖完整周期会导致样本不具有代表性。新奇效应和学习效应都会使短期数据不能反映长期稳态效果。

## 二、数据采集与校验阶段

### 2.1 数据完整性校验

**做什么：** 检查数据是否完整、是否有异常。

**校验项：**
- 分组与展示页面一致性（control必须看old_page，treatment必须看new_page）
- 用户去重（同一用户多次曝光只保留首次）
- 缺失值检查
- 时间戳完整性（必须有日期，不能只有时间）
- 样本量是否达到预设目标

**原理：**
- 不一致记录意味着分流系统有bug，分组可能不是真正随机的
- 重复曝光违反独立性假设，使标准误估计偏小，容易得到假阳性
- 缺少日期无法做时间趋势分析，无法排除周期效应

### 2.2 AA实验验证

**做什么：** 在正式实验前或同时，跑一组AA实验（两组都看旧页面），验证分流系统是否正常。

**方法：** 对AA数据做同样的z检验，预期p>0.05。如果AA实验显著，说明分流有问题。

**原理：** AA实验是验证随机化的金标准。如果两组看的是同一个页面，但转化率差异显著，只能说明两组用户本身就有差异——随机化失败了。此时AB实验的结论不可信。

## 三、分析阶段

### 3.1 描述性统计

**做什么：** 在做任何检验前，先"看"数据。

**内容：**
- 各组样本量、核心指标的均值/比例
- 分层统计（按国家、设备类型、新老用户等维度拆分）
- 检查随机化质量（两组样本量是否接近、各维度分布是否均衡）

**原理：** 描述性统计是"sanity check"——在用复杂方法之前，先用简单统计确认数据没有明显问题。分层统计还能提前发现可能的异质性处理效应（HTE），即实验效果在不同群体中可能不同。

### 3.2 假设检验

**做什么：** 用统计检验判断两组差异是否显著。

**方法选择：**

| 指标类型 | 检验方法 |
|---------|---------|
| 比例（转化率、点击率） | 双比例z检验 或 卡方检验 |
| 连续变量（客单价、停留时长） | 双样本t检验 或 Welch t检验（方差不齐时） |
| 非正态连续变量 | Mann-Whitney U检验 |
| 计数数据 | 泊松回归 或 负二项回归 |

**双比例z检验原理：**
1. H₀: p₁ = p₂
2. 计算合并比例 p̂ = (x₁+x₂)/(n₁+n₂)
3. 标准误 SE = √(p̂(1-p̂)·(1/n₁ + 1/n₂))
4. z = (p̂₁ - p̂₂) / SE
5. p值 = P(|Z| > |z|)

**关键：看置信区间，不只看p值**

p值只告诉你"是否显著"，置信区间告诉你"效应可能在什么范围"。面试时说"95%CI上界仅+0.08%，即使有效果也极小"比说"p=0.19不显著"有力得多。

### 3.3 辛普森悖论检查

**做什么：** 按关键分层变量拆分，检查各层差异方向是否与整体一致。

**方法：**
1. 识别可能的混杂变量（国家、设备类型、新老用户等）
2. 在每层内分别做假设检验
3. 比较每层的差异方向与整体方向

**原理：** 辛普森悖论的本质是混杂变量同时影响了分组分配比例和结果。当某层样本量占主导时，该层的效果方向会"压过"其他层的反向信号，导致整体结论与分层结论矛盾。

**发现悖论后的处理：**
- 不要用整体结论做决策
- 按层分别分析，使用回归/分层模型控制混杂
- 评估是否需要针对反转的亚组单独设计实验

### 3.4 辅助指标分析（以决策耗时为例）

**做什么：** 除核心指标外，分析辅助指标（如页面停留时长、决策耗时、点击深度等）是否改善，从多维度评估改版效果。

**方法：**
- **分布检验先行：** 用KS检验检查辅助指标是否满足正态假设，非正态则使用Mann-Whitney U等非参数检验
- **分层分析：** 区分转化用户与非转化用户的耗时含义不同——转化用户的耗时是"决策时间"，非转化用户的耗时只是"浏览时长"
- **交互项检验：** 在逻辑回归中加入 耗时×分组 交互项，检验新页面对转化的影响是否随耗时不同而不同
- **分桶分析：** 按耗时分桶看各组转化率，发现非线性关系

**原理：** 
- 核心指标（转化率）回答"效果是否更好"，辅助指标（决策耗时）回答"效率是否更高"。好的改版应该同时提升效果和效率。
- 非正态连续变量用t检验，虽然大样本下可用但不精确，非参数方法更稳健
- 交互项检验是发现"条件效应"的关键——新页面可能对快决策者有效但对慢决策者无效，这种异质性在整体均值中被平均掉
- 分桶分析能捕捉U型等非线性关系，避免逻辑回归中线性假设的局限

### 3.5 异质性处理效应分析（HTE）

**做什么：** 检验实验效果是否在不同用户群体中不同。

**方法：**
- 交互项检验：在回归中加 group × 特征 的交互项，检验交互项是否显著
- 分组回归：按维度拆分后分别估计处理效应
- CATE（条件平均处理效应）：用因果森林等机器学习方法估计异质性

**原理：** 整体平均效应可能掩盖了某些群体的正向效果和另一些群体的负向效果。发现HTE后可以：
- 对正向群体定向上线新方案
- 对负向群体保留旧方案
- 提升整体收益（精准策略 vs 一刀切）

## 四、决策阶段

### 4.1 综合评估框架

**不是"显著=上线，不显著=不上线"，而是多维度综合判断：**

| 判断维度 | 关键问题 |
|---------|---------|
| 统计显著性 | p值是否<0.05？ |
| 实际显著性 | 效应量是否有业务价值？CI是否排除无价值区间？ |
| 护栏指标 | 核心指标提升了，但其他指标是否恶化？ |
| 异质性 | 效果是否在所有群体一致？是否有群体受损？ |
| 功效 | 如果不显著，是因为真没效果还是功效不足？ |
| 数据质量 | 是否有缺失、不一致、周期效应等问题影响结论？ |

### 4.2 常见决策陷阱

**陷阱1：p-hacking（数据窥探）**
- 表现：反复检查数据，一显著就停；不显著就继续跑
- 危害：膨胀第一类错误率，假阳性率远超5%
- 正确做法：实验前确定样本量和运行周期，到点就停，不管显著与否

**陷阱2：把"不显著"等同于"没效果"**
- 表现：p>0.05就说"新方案无效"
- 危害：可能是功效不足导致的假阴性
- 正确做法：看置信区间——如果CI包含0但上界很大，说明可能有效果但不确定；如果CI上界很小（如+0.08%），说明即使有效果也微不足道

**陷阱3：只看核心指标忽视护栏**
- 表现：转化率提升了0.5pp就上线，但页面加载时间增加了2秒
- 危害：短期指标改善但长期用户体验恶化
- 正确做法：护栏指标恶化时即使核心指标显著也不上线

**陷阱4：辛普森悖论被忽视**
- 表现：整体看显著提升就上线，但某重要亚组实际上受损
- 危害：上线后部分用户体验下降，引发投诉
- 正确做法：分层检查，确认效果在各群体方向一致

### 4.3 决策输出模板

**推荐结论格式：**

1. **结论**：上线/不上线/继续实验/重新设计实验
2. **核心指标**：效应量、p值、95%CI
3. **护栏指标**：各指标变化情况
4. **异质性**：是否发现HTE，各亚组效果
5. **风险**：数据质量限制、可能的混杂
6. **后续建议**：如需继续实验，具体改进方案

## 五、进阶方法

### 5.1 多重比较校正

**场景：** 同时检验多个指标或多个分组时。

**方法：**
- Bonferroni校正：α' = α/k（k为检验次数），最保守
- Benjamini-Hochberg（FDR控制）：控制假发现率，比Bonferroni更powerful
- 正交分层：将指标分层（主要/次要/探索性），每层用不同α

**原理：** 每次检验有5%的假阳性概率，做20次检验，至少一次假阳性的概率 = 1-(0.95)^20 ≈ 64%。不校正的话，几乎必然出现"显著"结果。

### 5.2 溢出效应（Spillover Effect）

**场景：** 实验组用户的行为可能影响对照组用户（如社交网络中好友互相影响）。

**方法：**
- 网络聚类实验：按社交关系聚类，整簇分配
- 边界分析：比较实验组边缘用户（有对照组好友）vs 中心用户

**原理：** 标准AB实验假设个体处理效应独立（SUTVA假设）。如果存在溢出，对照组用户的行为被实验组"污染"，不再反映真实的控制效果，导致估计偏差。

### 5.3 方差缩减（Variance Reduction）

**场景：** 样本量有限，想提升功效但不增加流量。

**方法：**
- CUPED（Controlled-experiment Using Pre-Experiment Data）：用实验前的协变量调整，可削减50%+方差
- 协方差分析（ANCOVA）：在回归中加入实验前指标作为协变量
- 分层抽样：先按关键变量分层，层内随机分配

**原理：** 转化率的方差中，一部分是用户间的个体差异造成的，与实验无关。用实验前的数据（如历史转化率）作为协变量，可以"解释掉"这部分方差，使残差方差变小，等价于增大了有效样本量。

### 5.4 交错实验（Interleaving）

**场景：** 搜索排序、推荐算法等场景，用户可以同时看到两组的结果。

**方法：** 将A组和B组的结果混合展示给同一用户，比较用户对哪组结果的点击率更高。

**原理：** 每个用户同时作为自己的对照，消除了个体间差异，方差大幅降低，所需样本量可比AB实验少一个数量级。

## 六、AB实验流程速查图

```
实验设计 → 明确假设 → 定核心指标+辅助指标+护栏指标
         → 确定MDE → 计算样本量
         → 分流机制 → 哈希/正交/分层
         → 周期规划 → 覆盖完整周期+新奇效应缓冲期

数据采集 → AA实验 → 验证随机化
         → 一致性校验 → 分组与展示匹配
         → 去重/缺失值/时间戳检查

分析     → 描述性统计 → sanity check
         → 假设检验 → z/t/卡方 + 置信区间
         → 辅助指标 → 非参数检验+交互项+分桶分析
         → 辛普森悖论 → 分层检验方向一致性
         → HTE分析 → 交互项/分组回归
         → 功效验证 → 确认样本量是否达标

决策     → 综合评估 → 统计+实际+辅助指标+护栏+异质性+数据质量
         → 输出结论 → 上线/不上线/继续/重新设计
         → 后续追踪 → 上线后监控+长期效应评估
```
